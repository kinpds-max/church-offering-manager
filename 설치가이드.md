# 교회 재정실 헌금관리 시스템 - 설치 가이드

GitHub 저장소: https://github.com/kinpds-max/church-offering-manager

전체 구조: 데이터는 **구글 시트**에 저장되고, **Apps Script**가 API 역할을 하며, **웹사이트(index.html)**가 화면 역할을 합니다. 아래 순서대로 딱 한 번만 설정하면 계속 사용할 수 있습니다.

## 1단계. GitHub Pages로 웹사이트 켜기

1. GitHub 저장소 페이지 → **Settings** → 왼쪽 메뉴 **Pages**
2. Source: **Deploy from a branch** 선택
3. Branch: **main** / 폴더: **/ (root)** 선택 후 Save
4. 1~2분 후 `https://kinpds-max.github.io/church-offering-manager/` 주소로 접속 가능 (이 주소가 재정실에서 쓸 홈페이지 주소입니다)

## 2단계. 구글 시트 만들기

1. Google Drive에서 새 스프레드시트 생성 (예: "교회 헌금 관리")
2. 시트 이름은 자유롭게 지어도 됩니다. 탭(시트) 구성은 스크립트가 자동으로 만들어줍니다.

## 3단계. Apps Script 붙여넣기

**방법 A (기본): 시트에서 바로 열기**

1. 방금 만든 스프레드시트에서 상단 메뉴 **확장 프로그램 → Apps Script**
2. 기본으로 열려 있는 `Code.gs` 파일 내용을 모두 지우고, 저장소의 `Code.gs` 내용을 그대로 복사해서 붙여넣기
3. 저장(Ctrl+S)
4. 상단 함수 선택 드롭다운에서 **setupSheet** 선택 → 실행(▶) 버튼 클릭
5. "권한이 필요합니다" 팝업 → 계정 선택 → "고급" → "이동(안전하지 않음)" → 허용
   - (개인 프로젝트 스크립트라 나오는 정상적인 경고입니다)
6. 실행 후 스프레드시트로 돌아가면 `헌금기록`, `설정` 탭이 자동으로 생성되어 있습니다

**방법 B: Google Drive 오류로 "확장 프로그램 → Apps Script"가 안 열릴 때**

Drive가 일시적으로 불안정해서 위 방법이 안 되면, 시트를 거치지 않고 독립 스크립트로 만들 수 있습니다.

1. 주소창에 `https://script.google.com/` 입력해서 직접 접속
2. **새 프로젝트** 클릭
3. `Code.gs` 내용을 붙여넣기
4. 코드 상단의 `const SPREADSHEET_ID = '';` 부분에, 스프레드시트 주소의 `/d/`와 `/edit` 사이 문자열을 넣기
   - 예: 주소가 `https://docs.google.com/spreadsheets/d/1bxl_4J_UfhA855Utc-33A3kpoYy07uzxnVjJL0wXVRs/edit`라면
   - `const SPREADSHEET_ID = '1bxl_4J_UfhA855Utc-33A3kpoYy07uzxnVjJL0wXVRs';`
5. 저장(Ctrl+S) 후 함수 선택 드롭다운에서 **setupSheet** 실행 → 권한 승인
6. 스프레드시트로 가서 `헌금기록`, `설정` 탭이 생겼는지 확인

이후 4단계(웹 앱 배포)는 방법 A, B 동일합니다.

## 4단계. 웹 앱으로 배포

1. Apps Script 화면 오른쪽 위 **배포 → 새 배포**
2. 유형 선택(톱니바퀴) → **웹 앱**
3. 실행 계정: **나(Me)** / 액세스 권한: **전체(Anyone)** 선택
4. **배포** 클릭 → 권한 승인
5. 배포 완료 후 나오는 **웹 앱 URL**을 복사 (`https://script.google.com/macros/s/.../exec` 형태)

> 이후 Code.gs를 수정하면 "배포 → 배포 관리 → 수정(연필 아이콘) → 새 버전"으로 다시 배포해야 반영됩니다.

## 5단계. 웹사이트 첫 접속 설정

1. `https://kinpds-max.github.io/church-offering-manager/` 접속
2. 첫 화면에서 4단계의 **웹 앱 URL**과 **재정실 공용 비밀번호**(원하는 걸로 새로 정하기) 입력 후 "연결하기"
3. 이 비밀번호는 이후 재정실에서 계속 사용할 로그인 비밀번호가 됩니다 (스프레드시트 메뉴의 "헌금관리 → 비밀번호 재설정"에서 나중에 변경 가능)

## 사용법 요약

- **헌금입력 탭**: 날짜 선택 후 헌금종류별 금액 입력 → 저장. 같은 날짜를 다시 열면 입력된 내역과 삭제 버튼이 보입니다.
- **보고서 탭**: 주별 / 월별 / 연간 중 선택 → 조회. 결과는 화면에서 바로 확인, **CSV 다운로드**, **인쇄/PDF 저장**, **이메일로 받기**가 가능합니다.
  - 이메일로 받으려면 설정 탭에서 수신 이메일을 먼저 등록하세요.
- **설정 탭**: 헌금 종류 목록(쉼표로 구분) 수정, 보고서 수신 이메일 변경.

## 참고 사항 (보안)

- 이 시스템은 재정실 공용 비밀번호 하나로 접근을 제한하는 간단한 방식입니다. 은행 보안 수준은 아니므로, 웹 앱 URL과 비밀번호는 재정실 관계자에게만 공유하세요.
- 원본 데이터는 항상 여러분의 구글 시트에 그대로 남아 있고, 웹사이트는 그 시트를 읽고 쓰는 화면일 뿐입니다. 시트 자체를 백업/버전관리하고 싶다면 구글 드라이브의 "버전 기록" 기능을 사용하세요.
- 여러 사람이 같은 비밀번호로 입력하므로, 누가 입력했는지 구분하려면 "비고"란에 이름을 남기는 것을 권장합니다.

## 문제 해결

- **"unauthorized" 오류**: 비밀번호가 틀렸거나, 첫 연결 이후 시트 메뉴에서 비밀번호를 재설정한 경우 웹사이트에서도 "연결 해제" 후 새 비밀번호로 다시 연결하세요.
- **연결 실패**: 웹 앱 URL이 `/exec`로 끝나는지 확인하세요 (`/dev`가 아님). 배포 시 액세스 권한이 "전체(Anyone)"인지 확인하세요.
- **이메일이 안 옴**: 설정 탭에 수신 이메일이 등록되어 있는지 확인하세요.
